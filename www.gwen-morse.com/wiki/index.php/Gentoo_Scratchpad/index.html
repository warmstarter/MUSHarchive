Desktop Notes

  • nVidia uses GTK for its GUI stuff
  • KDE uses QT
  • LXDE uses C++ and gtk+
  • XFCE uses gtk+

LXDE uses the least resources. XFCE base uses slightly more resources but has the dock that I want. I ended up "rolling my own" desktop by emerging xfce and slim, and then the assorted lightweight xfce packages I like.

  • emerge xfce-meta slim

This meant giving up Dolphin and Kate for Thunar and Mousepad, but other than those two packages I don't miss anything.

Portage Maintenance

  • To estimate the time an emerge will take use -p for the emerge command, and -p for genlp: emerge world -uDNpv --with-bdeps=y | genlop -p
  • To skip a single specific package, use: "--exclude"

If you want old packages/flags freshened up (removes old packages) 

 emerge -DuN world
 emerge --depclean
 revdep-rebuild
 eix-test-obsolete

If you can't emerge something because you need to update config files: 

 etc-update

If online and you want to update mirrors: 

 mirrorselect -i -o >> /mnt/gentoo/etc/portage/make.conf

Hit 'spacebar' for each mirror you want.

To get multilib 

 emerge emul-linux-x86 packages OR set "abi_x86_32" flag for all libs that need it.

Lightweight KDE 

 emerge kdm kwin konqueror kdesktop

If you want 'everything' re-compiled (let run overnight): 

 emerge -e system && emerge -e world

Normal world emerge: 

 emerge -auvDN world

To rebuild after changing modules and use flags 

 emerge @module-rebuild
 emerge -uD --newuse @world
 revdep-rebuild

To rebuild and do a very thorough upgrade: 

 emerge -avuDt --newuse --changed-use --with-bdeps=y @world
 revdep-rebuild

Fixing up Perl: 

 perl-cleaner --reallyall

This will upgrade modules after kernel upgrade. 

 emerge @module-rebuild

This will upgrade X drivers after Xorg upgrade. 

 emerge @x11-module-rebuild
 emerge @module-rebuild && emerge -uD --newuse @world && revdep-rebuild

This will skip one package (example ncurses) 

 emerge -auvD --exclude=ncurses world

Grub Menu Updates

To boot into an ISO image (Rescue CD)

NOTE: In order to boot into an iso without burning it to a disk, you *must* make sure the ISO is stored in a partition that does not have a full linux filesystem. It can be stored in /boot, if /boot is a separate partition from root, or it can be stored on a separate disk.

I use a SD memory card that is always inserted into my computer and holds a number of "necessary" files.

After messing with this, I figured out the hardest part was determining where the kernel an initrd files may be hiding. This is specific to each distro. I mounted each ISO (using AcetoneISO) and looked through the various directories till I found the needed files.

  • Booting livecd-iso How to create a USB which boots multiple .ISO files using Grub2
  • How can I boot a squashfs image on my hard drive?
  • Boot Multiple ISO from USB via Grub2 Code 
menuentry "Windows 7" --class windows --class os {
    insmod part_msdos
    insmod ntfs
    set root='(hd0,msdos3)'
    chainloader +1
}

#!/bin/sh
exec tail -n +3 $0
### Add entries below this
set default=0
set timeout=8
insmod ext2
set root=(hd2,1)

menuentry "Kubuntu 16.04 ISO" {
set isofile="/boot/iso/kubuntu-16.04-desktop-amd64.iso"
loopback loop (hd2,1)$isofile
echo "Starting $isofile..."
linux (loop)/casper/vmlinuz.efi boot=casper iso-scan/filename=${isofile} quiet splash 
initrd (loop)/casper/initrd.lz
}
menuentry "Gentoo July 2016 Live DVD" {
set isofile="/boot/iso/livedvd-amd64-multilib-20160704.iso"
loopback loop (hd2,1)$isofile
linux (loop)/isolinux/gentoo root=/dev/ram0 init=/linuxrc dokeymap looptype=squashfs loop=/image.squashfs cdroot initrd=/isolinux/gentoo.xz isoboot=${isofile} keyboard=us
initrd (loop)/isolinux/gentoo.xz
}
menuentry "ARCH OpenRC June 2016 ISO" {
set isofile="/boot/iso/archlinux-openrc-2016.06.09-dual.iso"
loopback loop (hd2,1)$isofile
linux (loop)/arch/boot/x86_64/vmlinuz root=/dev/ram0 init=/linuxrc dokeymap looptype=squashfs loop=/image.squashfs cdroot initrd=/boot/initrd isoboot=${isofile} keyboard=us
initrd (loop)/arch/boot/x86_64/archiso.img
}

Kernel Updates

MAKEOPTS="-j3" make oldconfig && MAKEOPTS="-j3" make && MAKEOPTS="-j3" make modules_install && MAKEOPTS="-j3" make install

emerge genkernel gentoo-sources
MAKEOPTS="-j3" genkernel all
grub-install /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg
emerge @x11-module-rebuild
emerge @module-rebuild

If the kernel symlink gets broken: 

ln -sf /usr/src/linux-#.#.##-gentoo /usr/src/linux

The most useful grub and genkernel config files:

  • /etc/default/grub
  • /etc/genkernel.conf
  • /etc/grub.d/

Reset grub.cfg: 

grub-install /dev/sda && grub-mkconfig -o /boot/grub/grub.cfg

Locale Weirdness

Despite setting a correct locale variable, mine breaks.

To start, uncomment locale in /etc/locale.gen 

en_US ISO-8859-1
en_US.UTF-8 UTF-8

Next up, generate the locale settings: 

locale-gen

Then edit /etc/env.d/02locale and put the locales in by hand: 

LANG="en_US.utf8"
LC_CTYPE="en_US.utf8"
LC_NUMERIC="en_US.utf8"
LC_TIME="en_US.utf8"
LC_COLLATE="en_US.utf8"
LC_MONETARY="en_US.utf8"
LC_MESSAGES="en_US.utf8"
LC_PAPER="en_US.utf8"
LC_NAME="en_US.utf8"
LC_ADDRESS="en_US.utf8"
LC_TELEPHONE="en_US.utf8"
LC_MEASUREMENT="en_US.utf8"
LC_IDENTIFICATION="en_US.utf8"
LC_ALL=

Use eselect to write locale settings everywhere they need to be: 

eselect locale set 5

When all done, run: 

env-update && source /etc/profile

Once locale is properly set, you can re-emerge glibc.

Slot Conflict

I had a slot conflict that fixed itself (?). Here are notes on what to do if it needs to be manually cleared.

If "qt-core and qt-gui" are the packages in conflict, run this: 

emerge -C qt-core qt-gui; emerge -1 qt-core qt-gui

Clean out each conflict until you can "emerge -DuN world" again. Do not shut down before emerge runs cleanly.

In Case of Broken Ncurses

  1. Boot from Gentoo demo / recovery CD
  2. mkdir /mnt/gentoo
  3. mount /dev/hda3 /mnt/gentoo
  4. cp /lib/libncurses.so /mnt/gentoo/lib
  5. cp /lib/libncurses.so.5 /mnt/gentoo/lib
  6. reboot and emerge ncurses again

After re-emerging ncurses

  1. ln -s /lib/libncurses.so.5.2 /lib/libncurses.so.5
